# OctoAcme Project Management Docs

## Purpose
A central index and concise summary of OctoAcme's project management processes and links to the detailed process documents stored in this docs/ folder. This README helps new team members and stakeholders quickly find the right process guidance, onboarding materials, and checklists.

## Brief Overview of OctoAcme Project Management Processes
OctoAcme follows an iterative, customer-focused delivery model with clear ownership, measurable outcomes, and continuous improvement. Projects move through a lifecycle of initiation, planning, execution, release, and retrospective. Roles (Project Manager, Product Manager, Developers, QA) and artifacts (Project One-pager, backlog, risk register, release notes) ensure alignment and transparency. Teams use regular rituals (standups, planning, demos, retros) and runbooks/checklists to manage quality, risks, and releases.

## How to use these docs
- Keep the project one-pager and release notes updated in the project repo.
- Use the checklists inside process docs to validate readiness for planning, release, and post-mortem activities.
- Add or update process docs in this folder as the team refines practices; follow the repository's documentation contribution guidelines.
